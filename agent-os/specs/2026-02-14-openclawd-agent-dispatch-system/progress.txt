## Codebase Patterns

- **Python binary**: Use `python3` not `python` (python not found on this system)
- **DB path resolution**: `_get_db_path()` in `agent-dispatch/migrations.py` resolves via `OPENCLAWD_DB_PATH` env var with fallback to relative path from `agent-dispatch/` to `orchestrator-dashboard/orchestrator-dashboard/coordination.db`
- **Connection setup**: Always call `PRAGMA journal_mode=WAL` and `PRAGMA foreign_keys=ON` per connection (see `_get_connection()`)
- **Idempotent ALTER TABLE**: Use `try/except sqlite3.OperationalError` checking for "duplicate column name" - SQLite doesn't support `IF NOT EXISTS` for ALTER TABLE ADD COLUMN
- **Migration module pattern**: `agent-dispatch/migrations.py` contains all schema migrations with `migrate_*` functions and `verify_*` validation functions. Run via `python3 -m agent-dispatch.migrations`

  - `sqlite3.Row` row_factory enables dict-like access to query results
