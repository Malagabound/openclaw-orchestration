## Codebase Patterns

- ALTER TABLE ADD COLUMN migrations use try/except for "duplicate column name" to achieve idempotency
- All migrations follow pattern: `_get_connection()`, try/execute/commit, finally close
- `dispatch_db.run_migrations()` calls each migration function in order with db_path
- Migrations live in `agent-dispatch/migrations.py`, registered in `agent-dispatch/dispatch_db.py:run_migrations()`
- No npm/mypy/ruff available — verify syntax with `python3 -c "import ast; ast.parse(...)"`
- To modify CHECK constraints in SQLite, use table rebuild pattern (create temp with updated SQL, copy, drop, rename)
- Generated columns (GENERATED ALWAYS AS ... STORED) don't appear in PRAGMA table_info; use PRAGMA table_xinfo instead
- Testing agent-dispatch modules: use importlib to register `agent_dispatch` in sys.modules (directory has a hyphen). Register submodules AND setattr on parent for patch() to work. (Discovered in US-050)

- **compensate() function:**
  - Accepts task_id, agent_name, error_code, db_path
