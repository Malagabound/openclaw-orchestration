## Codebase Patterns

### Prompt File Structure
All agent prompt files follow this structure:
1. Identity (agent name, purpose)
2. Expertise (bullet list)
3. Domains (keyword list)
4. Allowed Tools (bullet list with descriptions)
5. SOP Workflow (Steps with Decision Gates)
6. Quality Standards (numbered list)
7. **Dispatch System Awareness** (new)
8. **Figure It Out** (new)
9. **Cross-Agent Handoff** (new)
10. Routing Keywords (final section)

### AGENT_SKILLS Key Naming
- Keys are functional role names: `research`, `product`, `comms`, `ops`, `meta`
- Prompt filenames match keys: `prompts/research.md`, `prompts/product.md`, etc.
- New entries: `haven`, `vault`, `george` (all with empty skill lists)
- Keys without persona files remain unchanged (`content`, `taskr`, `security`, `memory`)

### George's Adapter Methods
These are NOT universal tool-loop tools:
- Available in direct chat sessions via Python import
- No YAML definitions in tools/definitions/
- Cannot be called via standard tool_call mechanism without YAML definitions
- List: create_task, complete_task, add_task_contribution, log_agent_activity, create_notification, squad_chat_post, get_dashboard_summary, agent_checkin, determine_agents, get_coordination_summary
