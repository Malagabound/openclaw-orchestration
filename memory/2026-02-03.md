### Browser Testing Solution - COMPLETED ✅
**Problem:** Multiple Claude instances sharing same browser profile → collisions, "DevTools already open" errors

**Solution:** MCP Wrapper Script
- Created `scripts/playwright-mcp-wrapper.sh`
- Each instance gets unique profile: `/tmp/playwright-mcp-<PID>-<timestamp>`
- Auto-cleanup on exit
- Updated `.mcp.json` to use wrapper instead of direct npx

**Final Solution:** Use `--isolated` flag directly (no wrapper needed)
```json
"playwright": {
  "type": "stdio",
  "command": "npx",
  "args": ["@playwright/mcp@latest", "--isolated"],
  "env": {}
}
```

**Why this works:**
- `--isolated` creates in-memory browser profiles — no disk conflicts possible
- Each Claude instance spawns its own MCP process with its own browser
- Simpler than a wrapper script — official Playwright feature designed for this

**Database Naming Conventions (from OOS):**
- Tables: snake_case, plural (`finding_answer_conditions`)
- Columns: snake_case (`finding_rule_id`, `created_at`)
- Primary Key: `id` UUID with `gen_random_uuid()`
- Foreign Keys: `[table_singular]_id`
- Timestamps: `created_at`, `updated_at` TIMESTAMPTZ
- Soft delete: `deleted_at`
- Indexes: `idx_[table]_[columns]`
- RLS Policies: `[table]_[action]_[scope]`

**Frontend Naming Conventions:**
- Page components: PascalCase + Page suffix
- UI components: PascalCase
- Feature directories: kebab-case
- Hooks: camelCase with `use` prefix
- Types: PascalCase interfaces
- Type files: kebab-case
