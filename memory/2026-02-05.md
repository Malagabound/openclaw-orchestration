## QB Browser Automation Breakthrough

### Problem Solved
API cannot access "For Review" transactions - they're siloed in Plaid/Yodlee's domain. **Browser automation is the only path.**

### What Works
- **OpenClaw browser with `profile="openclaw"`** can view all 80 "For Review" transactions
- **JS evaluate technique** - refs expire too fast due to QB's dynamic page updates
- Must use DOM queries directly via `browser act kind=evaluate`

### Click Pattern for Posting Transactions
```javascript
// 1. Click row to expand dropdown
const row = document.querySelector('[data-testid="transaction-row-0"]');
row?.click();

// 2. Wait for dropdown, then click Post button
const postBtn = document.querySelector('button[aria-label="Post"]');
postBtn?.click();
```
