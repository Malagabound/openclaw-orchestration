# Recurring Revenue Software - Product Types

All product types to consider when researching software subscription opportunities. Don't default to "traditional SaaS" — scan ALL of these for opportunities.

---

## Product Type Directory

### 1. SaaS (Software-as-a-Service)

**What:** Web-based applications users access via browser  
**Customer:** End users (B2B or B2C)  
**Revenue Model:** Monthly/annual subscription  
**Examples:** Notion, Slack, Basecamp, Calendly

**Discovery Sources:**
- G2, Capterra (B2B)
- Product Hunt
- Indie Hackers

**Build Effort:** Medium-High  
**Competition:** High  
**Passive-Friendly:** ✅ Yes

---

### 2. API-as-a-Service

**What:** Backend services developers integrate into their products  
**Customer:** Developers, technical teams  
**Revenue Model:** Per-call, usage-based, or tiered subscription  
**Examples:** Stripe API, Twilio, Clearbit, Abstract API

**Discovery Sources:**
- RapidAPI marketplace
- GitHub trending
- Stack Overflow questions
- Developer forums

**Build Effort:** Medium  
**Competition:** Medium (varies by niche)  
**Passive-Friendly:** ✅ Yes

**See:** `API-PRODUCTS.md` for detailed methodology

---

### 3. Marketplace Actors/Scrapers

**What:** Automation tools hosted on scraping/automation platforms  
**Customer:** Developers, marketers, researchers  
**Revenue Model:** Pay-per-result (platform handles billing)  
**Examples:** Apify actors, Phantombuster automations

**Discovery Sources:**
- Apify Store (gaps, low-quality actors)
- Phantombuster marketplace
- User requests/issues on existing actors

**Build Effort:** Low-Medium  
**Competition:** Low  
**Passive-Friendly:** ✅ Yes (platform handles distribution)

**Platforms:**
| Platform | URL | Revenue Share |
|----------|-----|---------------|
| Apify | apify.com/store | 70-80% to creator |
| Phantombuster | phantombuster.com | Varies |

---

### 4. Platform Plugins/Apps

**What:** Extensions that add functionality to existing platforms  
**Customer:** Users of the host platform  
**Revenue Model:** Monthly subscription via platform billing  

**Platforms & Marketplaces:**

| Platform | Marketplace | Opportunity Level |
|----------|-------------|-------------------|
| Shopify | apps.shopify.com | High (huge market) |
| WordPress | wordpress.org/plugins | High (but crowded in basics) |
| Chrome | chrome.google.com/webstore | Medium |
| Slack | slack.com/apps | Medium |
| Discord | discord.com (bots) | High (underserved niches) |
| Figma | figma.com/community | Medium |
| VS Code | marketplace.visualstudio.com | Medium |
| Notion | notion.so/integrations | Growing |
| Airtable | airtable.com/marketplace | Medium |
| Zapier | zapier.com (integrations) | Medium |
| Linear | linear.app/integrations | Niche |
| Webflow | webflow.com/apps | Growing |

**Discovery Method:**
1. Browse marketplace by category
2. Sort by installs/reviews
3. Read negative reviews for gaps
4. Find abandoned but popular plugins
5. Identify niches with few options

**Build Effort:** Low-Medium  
**Competition:** Varies by platform  
**Passive-Friendly:** ✅ Yes

---

### 5. Bot-as-a-Service

**What:** Automated bots for messaging/community platforms  
**Customer:** Community managers, server owners, businesses  
**Revenue Model:** Freemium, per-server pricing, feature tiers  

**Platforms:**

| Platform | Bot Type | Market |
|----------|----------|--------|
| Discord | Server bots | Huge (150M+ users) |
| Telegram | Channel/group bots | Large |
| Slack | Workspace bots | B2B focused |
| WhatsApp | Business bots | Growing |
| Twitter/X | Automation bots | Restricted but exists |

**Discovery Sources:**
- top.gg (Discord bot listing)
- Discord server requests
- Telegram bot directories
- r/discordapp, r/Telegram

**What to look for:**
- Niche communities with specific needs
- Popular bots with poor UX
- Requested features not being built
- Abandoned bots with active users

**Build Effort:** Low  
**Competition:** Low in niches  
**Passive-Friendly:** ✅ Yes

---

### 6. Data Feeds/Subscriptions

**What:** Ongoing access to curated, aggregated, or proprietary data  
**Customer:** Businesses, researchers, developers  
**Revenue Model:** Monthly subscription for data access  
**Examples:** Stock data APIs, lead databases, industry benchmarks

**Types of data products:**
- Aggregated public data (cleaned/structured)
- Scraped and maintained datasets
- Proprietary data collection
- Enrichment data (add info to existing records)

**Discovery Sources:**
- Data.world, Kaggle (what's popular?)
- Industry forums (what data do people need?)
- "Where can I get [X] data?" questions

**Build Effort:** Medium  
**Competition:** Low if unique data source  
**Passive-Friendly:** ⚠️ Needs ongoing data maintenance

---

### 7. AI/ML Model Access

**What:** Hosted AI models others can call via API  
**Customer:** Developers, businesses needing AI capabilities  
**Revenue Model:** Per-call or subscription tiers  
**Examples:** OpenAI API, Replicate models, Hugging Face Inference

**Opportunity angles:**
- Fine-tuned models for specific industries
- Specialized models (legal, medical, technical)
- Cheaper alternatives to big providers
- Models with specific compliance (privacy, on-prem)

**Discovery Sources:**
- Replicate trending models
- Hugging Face spaces
- "Alternative to [expensive AI]" searches
- Industry-specific AI needs

**Build Effort:** Medium-High  
**Competition:** Growing fast  
**Passive-Friendly:** ✅ Yes once deployed

---

### 8. White-Label/Reseller Software

**What:** Software others rebrand and sell to their customers  
**Customer:** Agencies, consultants, resellers  
**Revenue Model:** Per-seat, per-client, or revenue share  
**Examples:** White-label CRM, website builders for agencies, reporting tools

**Why attractive:**
- B2B2B model — your customers do the selling
- Less direct marketing needed
- Higher price tolerance (business expense)

**Discovery Sources:**
- Agency forums (what do they resell?)
- "White label [tool type]" searches
- Consultant communities

**Build Effort:** High (needs to be brandable)  
**Competition:** Low in niches  
**Passive-Friendly:** ⚠️ Medium (agency support needs)

---

### 9. Mobile Apps with Subscriptions

**What:** iOS/Android apps with recurring billing  
**Customer:** Consumers or business users  
**Revenue Model:** Monthly/annual via App Store/Play Store (30% cut)  
**Examples:** Headspace, Calm, fitness apps, habit trackers

**Reality check:**
- 30% platform tax
- App store discovery is hard
- High competition in consumer categories
- Review/rating system can make or break you

**When it makes sense:**
- Companion to web SaaS
- Niche B2B tools (field workers, specific industries)
- Utility apps with clear ongoing value

**Build Effort:** High  
**Competition:** Very high (consumer), Medium (niche B2B)  
**Passive-Friendly:** ⚠️ App store updates required

---

### 10. Desktop Software Subscriptions

**What:** Downloadable software with subscription licensing  
**Customer:** Professionals, businesses  
**Revenue Model:** Monthly/annual license  
**Examples:** Adobe CC, Sketch, Figma desktop

**When it makes sense:**
- Needs local processing power
- Offline functionality required
- Professional tools with high willingness to pay

**Build Effort:** High  
**Competition:** Medium  
**Passive-Friendly:** ⚠️ OS updates require maintenance

---

### 11. Embedded Software/SDK Licensing

**What:** Code libraries/SDKs developers embed in their products  
**Customer:** Development teams  
**Revenue Model:** Usage-based or tiered by scale  
**Examples:** Auth0, Sentry SDK, analytics libraries

**Challenge:** Requires strong developer adoption and trust

**Build Effort:** High  
**Competition:** Medium  
**Passive-Friendly:** ⚠️ Needs docs, support, updates

---

### 12. Membership Sites with Software Access

**What:** Software features gated behind membership  
**Customer:** Professionals, hobbyists in specific niche  
**Revenue Model:** Monthly membership fee  
**Examples:** SEO tool memberships, research platforms, trading tools

**Combines:**
- Content (tutorials, guides)
- Community (forums, Discord)
- Software (tools, calculators, generators)

**Discovery Sources:**
- Successful info-product creators who could add software
- Niches where spreadsheets are used but dedicated tools don't exist

**Build Effort:** Medium  
**Competition:** Medium  
**Passive-Friendly:** ✅ Yes

---

### 13. Infrastructure/DevOps Tools

**What:** Developer infrastructure and monitoring  
**Customer:** Engineering teams  
**Revenue Model:** Usage-based or tiered  
**Examples:** Vercel, Railway, LogRocket, Sentry

**Reality check:**
- Competitive space
- Needs strong technical moat
- High reliability expectations

**Build Effort:** Very High  
**Competition:** High  
**Passive-Friendly:** ❌ No (uptime critical)

---

## Priority Matrix

**Actively scan for opportunities in:**

| Priority | Product Types | Why |
|----------|---------------|-----|
| **High** | Marketplace Actors, Platform Plugins, Bots | Low competition, built-in distribution |
| **High** | API-as-a-Service | Developer audience finds you |
| **Medium** | SaaS (micro, niche) | Core competency, but competitive |
| **Medium** | Data Feeds, AI Models | If we find unique angle |
| **Medium** | Membership + Software | Combines content + tools |
| **Lower** | White-Label | Higher effort, agency support |
| **Lower** | Mobile, Desktop | Platform taxes, high effort |
| **Avoid** | Infrastructure | Too competitive, uptime pressure |

---

## Research Checklist

When scanning for opportunities, check ALL high-priority types:

- [ ] **Apify Store** — gaps in scraper/automation actors?
- [ ] **RapidAPI** — underserved API categories?
- [ ] **Shopify Apps** — merchant pain points?
- [ ] **WordPress Plugins** — abandoned popular plugins?
- [ ] **Chrome Extensions** — productivity gaps?
- [ ] **Discord Bots** — niche community needs?
- [ ] **Telegram Bots** — automation opportunities?
- [ ] **Product Hunt** — traditional SaaS launches?
- [ ] **Indie Hackers** — revenue-validated products?
- [ ] **G2/Capterra** — B2B software gaps?

Don't stop at the first category. A problem might be best solved as a Shopify app, not a standalone SaaS.
