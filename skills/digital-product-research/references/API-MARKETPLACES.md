# API & Code Marketplaces

Platforms where developers sell APIs, scrapers, automation tools, and code snippets. These are digital products with recurring/usage-based revenue potential.

---

## Why This Matters

- 62% of organizations now work with income-generating APIs
- 21% derive over 75% of total revenue from APIs
- Developer tools often command premium pricing
- Usage-based revenue = passive income potential

---

## Scraper/Automation Marketplaces

### 1. Apify Store

**URL:** https://apify.com/store
**What sells:** Web scrapers, automation actors, data extraction tools

**Revenue model:**
- Pay-per-result or flat monthly rental
- Apify takes 20% margin
- You set your own pricing

**Data available:**
- Total users
- Total runs
- Success rate
- Reviews/ratings

**Research method:**
1. Browse store by category
2. Sort by "Most users" or "Most popular"
3. Look for categories with few/poor options
4. Read reviews for feature gaps

**Opportunity signals:**
- Categories with only 1-2 scrapers
- Existing scrapers with poor reviews
- Marketplaces without dedicated scrapers (Creative Market, AppSumo)

## API Marketplaces

### 2. RapidAPI

**URL:** https://rapidapi.com
**What sells:** Any API — data, AI/ML, finance, utilities, etc.

**Stats:**
- 4+ million developers
- 35,000+ APIs listed
- Largest API marketplace

**Revenue model:**
- 20% commission (highest in industry)
- You set tiered pricing (Free/Basic/Pro/Ultra/Mega)
- Platform handles billing, auth, docs

**Research method:**
1. Browse categories: https://rapidapi.com/hub
2. Check "Popular" and "Newest" APIs
3. Look at pricing tiers and subscriber counts
4. Read API reviews for gaps

**Good for:** Maximum reach, zero infrastructure needed

---

### 3. APILayer

**URL:** https://apilayer.com
**What sells:** Curated, high-quality APIs (AI/ML, finance, geolocation)

**Stats:**
- 75+ curated APIs
- Quality-over-quantity approach

**Revenue model:**
- 15% commission (better than RapidAPI's 20%)
- Strict curation — must pass technical assessment

**Research method:**
1. Browse categories
2. Note what's missing vs RapidAPI
3. Curated = less competition but higher bar

**Good for:** Better economics, quality positioning

## Research Process for API/Code Products

### Discovery
```
1. Browse Apify Store — what scrapers are missing or poor?
2. Browse RapidAPI categories — gaps in data APIs?
3. Check CodeCanyon bestsellers — what's selling?
4. Look at Chrome extensions with poor ratings but high installs
```

### Validation
```
1. How many users/installs does competition have?
2. What do negative reviews complain about?
3. Is there a clear gap we could fill?
4. Can Alan build this in 1-2 weeks?
```

### Revenue Estimation
```
For Apify actors:
- Check similar actors' usage stats
- Estimate: [users] × [runs/user/month] × [price/run]

For APIs:
- Check RapidAPI subscriber counts on similar APIs
- Estimate: [subscribers] × [avg tier price]

For one-time code:
- Check CodeCanyon/Gumroad sales counts
- Multiply by price
```

## Key Metrics to Track

When evaluating API/code product opportunities:

- **Users/Installs** — Demand signal
- **Revenue per user** — Monetization potential
- **Churn/retention** — Stickiness
- **Support burden** — Ongoing cost
- **Competition count** — Market saturation
- **Review complaints** — Differentiation opportunities
